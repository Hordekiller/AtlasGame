# اطلس | AtlasGame 🏛️

> یک بازی استراتژیک شهرسازی و امپراتوری‌سازی الهام گرفته از Ikariam — ساخته شده با **Godot 4.7**

---

## وضعیت پروژه — ارزیابی صادقانه

| جنبه | وضعیت | توضیح |
|------|-------|-------|
| **بازی‌پذیری** | ⚠️ ناقص | هسته اصلی کار می‌کند اما برای تجربه کامل ناقص است |
| **پایداری** | ✅ خوب | خطای کامپایل ندارد، کرش غیرمنتظره ندارد |
| **UI/UX** | ⚠️ متوسط | پنل‌ها باز می‌شوند اما جریان کاربری کامل نیست |
| **Assets** | ⚠️ ناقص | ۵۴ تکسچر ساختمان، ۱۰ آیکون منبع، ۷ تکسچر یونیت |
| **محتوا** | 🔴 کم | هیچ دشمن NPC، سناریو، یا لول‌بندی وجود ندارد |
| **چندنفره** | 🔴 پیاده‌سازی نشده | هیچ کد شبکه‌ای وجود ندارد |

## ✨ ویژگی‌های پیاده‌سازی شده

### 🏙️ شهرسازی
- ۳۸ نوع ساختمان با تعاریف کامل (هزینه، تولید، مصرف، پیش‌نیاز تحقیق)
- سیستم کارگر (تخصیص نیروی انسانی به ساختمان‌ها)
- گرید شهری ۱۶×۱۶ با امکان چیدمان آزاد
- ۵۴ تکسچر ساختمان در `Assets/Textures/Buildings/`
- سیستم سازنده (Builder) با ۲ اسلات همزمان (الهام از Clash of Clans)
- ارتقا و تخریب ساختمان + progress bar بصری
- تایمر ساخت real-time (ثانیه‌ای)

### 📈 اقتصاد
- ۱۳ منبع (چوب، مرمر، شیشه، شراب، طلا، غذا، سنگ، کریستال، گوگرد، جمعیت، کارگر، امتیاز پژوهش، رضایت)
- سیستم رضایت (Happiness) با تأثیر شراب، میخانه، معبد، ازدحام و فساد
- سیستم فساد (Corruption) و کاهش با اقامتگاه فرماندار
- انبار و ذخیره‌سازی با محدودیت ظرفیت
- تخفیف هزینه ساختمانی (نجار، معمار، عینک‌ساز و...)
- تجارت بین شهری با مسیرهای دریایی (Travel time محاسبه می‌شود)
- تیک اقتصادی real-time (یک بار در ثانیه)

### 🔬 تحقیق
- ۴ شاخه اصلی: اقتصاد، نظامی، علم، دریانوردی
- فناوری‌های لایه‌ای با پیش‌نیاز (حدود ۳۰+ فناوری در ResearchManager)
- تولید امتیاز پژوهش با آکادمی و کارگاه کریستال
- UI کامل ResearchPanel
- آنلاک قابلیت‌های جدید با تحقیق

### ⚔️ نبرد (Combat System)
- سیستم نبرد نوبتی با ۱۲ راند (هر راند ۵ دقیقه = حداکثر ۱ ساعت نبرد)
- ۶ نوع فورمیشن (استاندارد، فالانژ، جناحی، یورش، سنگر، کمین)
- Commander modifiers: حمله، دفاع، دریایی
- BattleScene بصری با نمایش attacker/defender و timer شمارش معکوس
- دکمه عقب‌نشینی در راندهای ۳، ۶، ۹
- CombatReport با تفکیک per-round (تلفات، مهارت‌ها، غارت)
- DefensiveBuildings سیستمی (برج دیده‌بانی، توپ، زنجیره بنادر، طاق، دیوار)

### 🧑‍✈️ Commander System (الهام از Rise of Kingdoms)
- ۶ فرمانده: ناخدا، جنگ‌سالار، تاکتیسین، معمار، جاسوس، دریادار
- ۳ نوع (دریایی، زمینی، پشتیبانی) و ۴ ندرت (معمولی تا افسانه‌ای)
- ۳ مهارت per commander (passive + active)
- سطح‌بندی تا ۵۰ با EXP، آنلاک مهارت در Tiers 1-5
- CommanderPanel: انتخاب، مشاهده آمار و مهارت‌ها
- اتصال modifiers به CombatSystem

### 🌍 جهان جزیره‌ای
- ۲۰ جزیره با منابع اصلی و ثانویه
- تولید جهان deterministic (seed=42)
- نقشه جهان با رسم Canvas (جزایر، شهرها، مسیرهای تجاری)
- کلیک روی جزیره برای مشاهده یا استعمار
- Zoom/Pan با ماوس و لمس (Pinch zoom + Pan gesture)
- سیستم Trade Routes بین شهری

### 🕵️ جاسوسی
- ۶ نوع مأموریت (خرابکاری، سرقت منابع، سرقت تحقیق، شناسایی، شورش، مسموم کردن آب)
- مخفیگاه با قابلیت ارتقا
- ریسک کشف و اعدام
- پیاده‌سازی کامل در SpySystem.gd

### 📱 پلتفرم موبایل
- Landscape-only (پشتیبانی از چرخش اجباری به حالت افقی)
- SafeAreaManager برای notch/punch-hole دوربین
- HapticManager برای بازخورد لمسی (اندروید)
- Touch targets ≥ 88px
- AdvisorBar عمودی با ۸ دکمه (مناسب شست راست)
- Pinch-zoom و Pan gesture در WorldMap
- NotificationManager برای نوتیفیکیشن‌های محلی اندروید
- ETC2 فشرده‌سازی تکسچر (موبایل)
- Immersive mode + Keep screen on
- Beginner Protection (۷۲ ساعت محافظت)

### 🎁 سیستم‌های روزانه
- QuestSystem با ۶ ماموریت روزانه (اولین ساخت، اولین ارتقا، جمعیت ۵۰، تحقیق اول، آموزش ارتش، ورود روزانه)
- EventSystem با ۴ رویداد دوره‌ای (جشن برداشت، تمرین نظامی، بادهای تجاری، الهام)
- DailyReward (۷ روز چرخشی)

### 🎨 فنی
- معماری Signal-based (EventBus برای ارتباط بین سیستم‌ها)
- ObjectPool برای مدیریت tiles
- ResponsiveLayout با سایز کاشی داینامیک (۴۰-۷۲px)
- UITheme اختصاصی با StyleBoxFlat
- Shaderهای سفارشی (آب، glow، ذرات آتش، post-process bloom)
- Save/Load با Dictionary serialization
- TimeManager با tick/day سیستم و offline progress
- ۹٬۵۴۵ خط GDScript + ۱٬۷۷۸ خط TSCN

## ❌ کمبودها و موارد ناقص

### بحرانی

| مورد | توضیح |
|------|-------|
| **بدون دشمن NPC** | هیچ شهر دشمن یا باربارینی در جهان وجود ندارد. تمام شهرها فقط توسط بازیکن ساخته می‌شوند |
| **بدون سناریو/Tutorial** | هیچ راهنمای شروع بازی وجود ندارد. بازیکن تازه‌کار نمی‌داند چه کند |
| **بدون جریان گیم‌پلی کامل** | از MainMenu به Game می‌رود اما حلقه کامل بازی (ساخت → رشد → جنگ → پیشرفت) وصله نشده |
| **BattleScene ظاهری است** | نبرد به صورت ساده نمایش داده می‌شود، هیچ انیمیشن projectile یا unit وجود ندارد |
| **تکسچرهای گمشده** | ساختمان‌های دفاعی (توپ، برج، زنجیره بندر) و بسیاری از یونیت‌ها تکسچر ندارند و fallback به construct.png می‌خورند |

### متوسط

| مورد | توضیح |
|------|-------|
| **MilitaryPanel UI دارد اما flow ندارد** | پنل نظامی باز می‌شود اما آموزش سرباز و اعزام به جنگ وصله نشده |
| **TradePanel UI دارد اما بازار کار نمی‌کند** | دکمه‌ها و لیست وجود دارد اما تجارت با NPC یا بازیکنان پیاده‌سازی نشده |
| **AllyPanel استاب است** | فقط یک پنل خالی با بستن |
| **ColonizeDialog قلاب نشده** | دیالوگ استعمار جزیره در WorldMap صدا زده می‌شود اما ممکن است flow کامل نباشد |
| **ResearchManager به Military متصل نیست** | تحقیق یونیت‌ها را آنلاک می‌کند اما MilitaryManager از آن استفاده نمی‌کند |
| **EconomyManager production tick دارد اما consumption کامل نیست** | فقط غذا و شراب مصرف می‌شود |
| **HUD ساختمان جدید را نشان نمی‌دهد** | بعد از ساخت، HUD اطلاعات را به‌روز نمی‌کند تا کاربر rebuild کند |
| **ObjectPool هرگز prewarm نشده** | پول تعبیه شده اما در ابتدای بازی tiles از قبل ساخته نمی‌شوند |
| **NotificationManager اندروید schedule دارد اما اتصال ندارد** | نوتیفیکیشن‌ها هرگز در جریان بازی trigger نمی‌شوند |

### جزئی

| مورد | توضیح |
|------|-------|
| ۸ ساختمان از ۳۸ تکسچر اختصاصی ندارند (fallback به construct.png) |
| CityView `_ready()` rebuild_all() اخیراً اضافه شده،可能 at race condition با signalها |
| ذخیره/بارگذاری کامل تست نشده (مخصوصاً指挥官 و سیستم‌های جدید) |
| Offline progress در TimeManager ticks را محاسبه می‌کند اما ساختمان‌ها را کامل نمی‌کند |
| CitizenSystem اسپرت citizen دارد اما هیچکس در شهر راه نمی‌رود |
| Background شهر (`Bg`) پوزیشن فیکس دارد (484,767) و responsive نیست |
| Documentation (این README) کامل‌تر از خود بازیه |
| هیچ تست واحد (unit test) وجود ندارد |
| BattleSystem.gd قدیمی کنار گذاشته شده (جایگزین CombatSystem.gd شده اما فایل حذف نشده) |

```
│   └── Textures/            # تمام تصاویر بازی (۱۱۲۲ فایل)
│       ├── Buildings/       # ۵۴ اسپرایت ساختمان
│       ├── Resources/       # ۱۰ آیکون منبع
│       ├── Units/           # ۷ اسپرایت یونیت
│       ├── UI/              # ۳۲ المان UI
│       └── Environment/     # بک‌گراند شهر و جزیره
├── Globals/                 # ۴ اسکریپت سراسری
│   ├── Globals.gd           # تمام ثابت‌ها و enums
│   ├── GameState.gd         # وضعیت runtime بازی
│   ├── EventBus.gd          # سیستم signal (۴۴ event)
│   └── UITheme.gd           # تم و استایل UI
├── Scenes/
│   ├── Menu/                # منوی اصلی، تنظیمات، لود/ذخیره
│   ├── Game/                # ۲۰ فایل (HUD، CityView، WorldMap، پنل‌ها)
│   └── Building/            # کامپوننت بصری هر ساختمان
└── Scripts/
    ├── Core/                # ۱۰ فایل (Building, Economy, Time, World, Save, ...)
    ├── Military/            # MilitaryManager
    ├── Systems/             # ۱۰ فایل (Combat, Commander, Quest, Event, ...)
    └── Utils/               # ObjectPool
```

```bash
# نیاز: Godot Engine 4.7+
```

- کلیک چپ: انتخاب/قرار دادن ساختمان
